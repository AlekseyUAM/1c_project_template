
# Шаблон 1C проектов

Основная структура для разработки 1С конфигураций

## 📋 Требования

- **Visual Studio Code** (VS Code) — редактор кода
- **Docker** — средство контейнеризации
- **Git** — система контроля версий

## 📁 Структура каталогов

```
project_template/
├── src/                           # Исходный код
│   ├── cf/                        # Основная конфигурация (Configuration)
│   └── cfe/                       # Расширения конфигурации (Extensions)
│       └── YAXUNIT/               # Тестовый фреймворк YAX Unit
│       └── unit_tests/            # Расширение для юнит тестов
│       └── mcp_tools/             # расширение для 1C MCP Tools 
├── configs/                       # Конфигурационные файлы (DockerCompose, etc.)
├── docs/                          # Документация проекта
├── specs/                         # Спецификации требований
├── tasks/                         # Описания задач и работ
├── logs/                          # Логи сборки и выполнения
├── tmp/                           # Временные файлы (не коммитить)
├── .claude/                       # Конфигурация для Claude Code
│   └── settings.local.json        # Локальные настройки (не коммитить)
├── .gitignore                     # Правила игнорирования файлов для git
├── README.md                      # Этот файл
├── INSTALL.md                     # Инструкции по установке
├── .1c-devbase.json.example       # Пример файла с настройками для 1c-batch
├── .mcp.json.example              # Пример файла с настройками mcp
└── CLAUDE.md                      # Указания для Claude Code
```

## ⚙️ Установка и первый запуск

### Перед началом работы

1. **Клонируйте проект**
   ```bash
   git clone <репозиторий> project-name
   cd project-name
   ```
2. **Установите зависимости**

   Полная инструкция находится в [INSTALL.md](./INSTALL.md)

### После установки

1. **Проверьте конфигурацию Git**
   - Убедитесь, что `.gitignore` содержит нужные исключения
   - Выгрузите конфигурацию из 1C в файлы в каталог src/cf

2. **Проверьте локальные настройки Claude**
   - При необходимости отредактируйте `.claude/settings.local.json`

3. **Проверьте mcp серверы**
    ```bash
   /mcp
    ```
4. **Проверьте skills**
    ```bash
   /skills
    ```
5. **Проверье сборку проекта, запуск тестов через claude**
   ```bash
   "собери проект", "запусти тесты"
   ```
